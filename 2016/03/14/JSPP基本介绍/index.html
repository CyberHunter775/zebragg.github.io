JSP 介绍

  • JSP：Java Server Pager,是 Sun 开发的用于动态开发Web的技术，JSP 实际就是Servlet，jsp引擎编译之后为一个 Java 文件，查看源码最终继承的父类是 HttpServlet
  • JSP 就像写 HTML 代码，html 只能为用户提供静态数据，而JSP 技术允许在 页面中嵌套 Java 代码，为用户提供动态数据

JSP基本语法

  • JSP模板元素
    • JSP 页面中的HTML内容叫为 JSP 模板元素
    • JSP 模板元素定义了网页了基本框架

  • JSP脚本表达式

    • JSP脚本表达式用于将出现数据输出到客户端的语法：<%= 变量或表达式 &> 如当前的时间 ： <%= new java.util.Date() %>
    • JSP 引擎在翻译脚本表达式时，会将出现数据转成字符串，然后再相应的位置用 out.println() 将数据输给客户端
    • JSP 脚本中只能出现 Java 代码，不能出现其他模板元素，JSP 引擎在翻译 JSP 页面中，会将 JSP 脚本中的 Java 代码原封不动地放到 Servlet 的 _jspService 方法中
    • 一个 JSP 页面可以有多个脚本片段，多个脚本片段中的代码可以以互相访问

    • 单个脚本中的 Java 代码可以是不完整的，但是，多个脚本片段组合后的结果必须是一个完整的 Java语法 例如：

      <table border=1 width=200>
    <% 
        for(int i=0;i<5;i++){
         %>
         <tr>
             <td><%= i+"" %></td>
             <td><%= "aa" %></td>
         </tr>
         <%
        } 
     %>
</table>

  • JSP 声明

    • JSP 页面中编写的所有代码，默认会翻译到 serlvet 中的 service 方法中，Jsp 声明的 Java代码将会被翻译到 _jspService 方法中来
    • JSP 声明可以用来定义 JSP 页面转换成 的Servlet 程序中的 静态代码块、成员变量和方法
    • 多个 静态代码块、变量和函数可以定义在一个 JSP声明中，也可以分别定义在多个 JSP声明中
    • JSP 隐式对象的作用 范围仅限 于 Servlet 的 _jspService 方法，所以在 JSP 声明中不能使用这些隐式对象

  • JSP 指令

    • JSP 指令 是为了JSP引擎而设计的，他们并不直接产生任何可见输出，而只是告诉引擎如何处理 JSP 页面中的其余部分，JSP 2.0 规范中定义了三个指令：page、include、tablib

    • 基本语法

      • <%@ 指令 属性名 = “值” %> 例如：<%@ page contentType=”text/heml;charset=utf-8”%>

      • 指令中如果有多个属性，可以写在一个指令中，也可以分开写如:

        <%@ page contentType=”text/html;charset=gb2312”%>
        <%@ page import=”java.util.Date”%>
        <%@ page contentType=”text/html;charset=gb2312”;import=”java.util.Date”%>

    • page 指令
      • 用于定义 JSP页面中的各种属性，无论出现在 JSP 页面的什么位置处，作用的都是整个 JSP 页面，所以一般放在页面的起始位置
      • 中文乱码问题：Jsp 引擎将 JSP 页面翻译成 Servlet 源文件时也可能导致中文乱码问题，JSP 引擎将 JSP 源文件翻译成 Servlet 源文件默认常用的 ISO08859-1编码，而JSP 开发人员可以采用各种字符集来编码，因此，JSP 引擎将 JSP 源文件翻译成Servlet源文件时，需要进行字符。如果没有说明，JSP 默认作为 ISO8859-1来处理
      • page 指令的 contentType 属性说明 JSP 源文件的 字符集编码，page 指令的 pageEncoding 属性说明JSP 源文件的字符集编码
    • include 指令
      • 用于引入其他 JSP 页面，如歌使用 include 指令引入其他 JSP 页面，那么 JSP 引擎会将这2个 JSP 翻译成一个 servlet，所以 include 指令也叫作 静态引入
      • <%@ include file=”引入文件的URL”%> file 属性 用于指定被引入文件的 路径。路径如果以 “/” 开头，那么代表当前 web 应用
      • 注意 被引入的 文件必须遵循 JSP 语法 ； 被引入的文件可以使用任意的 扩展名，即使扩展名 是html，JSP 引擎也会安装处理 jsp 页面的方式处理它里面的 内容；使用 include 指令会涉及到 2 个 JSP 页面，并会把 2个JSP 翻译成 Servlet，所以这 2 个 JSP 页面的指令不能冲突
    • taglib指令
      • 用于在 JSP 页面中导入标签库

JSP 运行原理以及九大隐式对象

  • JSP 页面在第一次被访问的时候，WEB容器会把请求交给 JSP 引擎（java 程序） 去处理，JSP 引擎先将 JSP 翻译成一个 servlet，然后按照 Servlet的调用去处理
  • 由于第一次 JSP 访问时会被翻译成 Servlet，所以第一次访问的时候通常会比较慢，但是第二次访问的时候，如果 JSP 引擎发现 JSP 没有变化，就不再进行翻译了，直接调用
  • JSP 引擎在调用 JSP 对应的 Servlet 时，会传递或是 创建 9 个与 web 开发相关的对象 供 Servlet 使用，JSP 技术为开发人员在编写 JSP 页面时获得这些 web 对象的 引用，特意定义了 9 个对应的 对象，开发人员就可以再 JSP 页面中通过这些 变量来 快速获得 9 大对象的 引用
  • 9大对象分别为 JSP 的隐式对象：request、response、config、application、exception、Session、page、out、pageContext

JSP标签

  • JSP 标签也叫 JSP Action,用于在 Jsp 页面 中提供业务逻辑功能，避免JSP 页面中直接编写 Java 代码，造成 JSP 页面 难以维护
    • 用于把 另一个 资源的 输出 内容插入进 当前 的 JSP 页面的输出内容中，这种在JSP页面执行时引入的方式叫做 动态引入
    • 与 include 指令比较：
      • 前者 是动态引入的，涉及到 2个 JSP 页面 会被翻译成 2个 servlet，这2个servlet的内容在执行时会进行合并，代码级别的包含，将目标代码直接拷贝进来，这里要注意的是 不要写重复的
      • include 指令是静态引入的，涉及到的2个 JSP 页面会被翻译成一个servlet，内容是在源文件级别进行合并，执行目标页面，将结果包含进来
      • 都会把2个JSP 页面 内容合并输出，所以这2个页面不要出现重复的 HTML 全局架构标签，否则输出给客户端的 内容将会是混乱的 HTML 文档
    • 用于把一个请求转发给另外一个资源

    • 当使用 和 标签引入或将请求转发给其他资源的时候，可以使用 标签来向这个资源传递参数

    • 示例：

      <jsp:forward page="relativeURL | <%=expression%>">
     <jsp:param name="parameterName" value="parameterValue|<%= expression %>" />
                    name 用来指定参数名 value 用来指定参数值
                    可以使用多个<jsp:param> 标签来传递多个参数
 </jsp:forward>

<jsp:include page="relativeURL | <%=expression%>">
      <jsp:param name="parameterName" value="parameterValue|<%= expression %>" />
  </jsp:include>
